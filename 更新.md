# Starward 预下载与更新实现逻辑梳理

> 目标：提取 Starward 中“更新检测 / 自身更新 / 游戏预下载 / 游戏更新”的核心实现逻辑，整理成与具体编程语言无关的流程规范，方便用其他语言复现。

---

## 1. 核心概念与模块划分

Starward 的更新体系分为两条独立主线：

1. **启动器自身更新（Self Update）**
   - 由主程序直接完成（便携版）或交给安装包程序完成（Setup 版）。
   - 依赖一个“发布清单（Release Manifest）”描述新版本的所有文件。

2. **游戏预下载与更新（Game Pre-download / Update）**
   - 由主程序发起，实际下载/解压/合并交给独立的 RPC 工作进程执行。
   - 依赖游戏厂商（HoYoPlay / 米哈游）提供的游戏包元数据。

两种主线都复用同一套底层能力：

- **HTTP 断点续传下载**
- **文件哈希校验（MD5 / SHA256）**
- **差异补丁合并（HPatch / Zstd / 7z）**
- **重试与错误恢复**
- **任务状态机**

## 2. 更新检测逻辑

### 2.1 输入

- `currentVersion`：当前程序版本号
- `ignoreVersion`：用户手动忽略的版本号
- `enablePreview`：是否启用预览版通道
- `arch`：当前系统架构（x64 / arm64）
- `installType`：安装类型（Portable / Setup）

### 2.2 流程

```
function CheckUpdate(currentVersion, ignoreVersion, enablePreview, arch, installType):
    releaseInfo = FetchReleaseInfo(enablePreview, currentVersion)
    // releaseInfo 包含多个平台/安装类型的 ReleaseDetail
    detail = SelectByArchAndType(releaseInfo, arch, installType)

    latestVersion = ParseVersion(detail.version)
    cur = ParseVersion(currentVersion)
    ign = ParseVersion(ignoreVersion)

    if latestVersion > cur:
        if latestVersion > ign:
            return detail   // 需要更新
    return null             // 不需要更新
```

### 2.3 关键点

- 版本号比较必须支持 `1.2.3`、预览版（如 `1.2.3-beta`）等语义化版本规则。
- 忽略版本不是“永久不更新”，而是“不提示这个版本”；当比忽略版本更新的版本出现时，仍然提示。
- `FetchReleaseInfo` 通常从发布服务器拉取 JSON，包含版本号、清单 URL、安装包 URL、差量包信息等。

## 3. 启动器自身更新（便携版）

便携版要求可以直接覆盖自身文件，因此更新流程是“边运行边下载到缓存，完成后重启”。

### 3.1 流程概览

```
function SelfUpdatePortable(releaseDetail, targetPath, cacheFolder):
    manifest = FetchManifest(releaseDetail.manifestUrl)
    // 如果存在从 currentVersion 到 targetVersion 的差量清单，优先使用
    if releaseDetail.diffs contains currentVersion:
        manifest = FetchManifest(releaseDetail.diffs[currentVersion].manifestUrl)

    currentFiles = BuildFileHashMap(targetPath)   // 当前目录所有文件的哈希映射
    CreateDirectory(cacheFolder)

    BackupVersionIni(targetPath, cacheFolder)
    try:
        DownloadNeededFiles(manifest, currentFiles, cacheFolder)
        PatchAndVerifyFiles(manifest, currentFiles, targetPath, cacheFolder)
        DeleteObsoleteFiles(manifest, targetPath)
        DeleteDirectory(cacheFolder)
        return Success
    catch OperationCanceled:
        RestoreVersionIni(cacheFolder, targetPath)
        return Canceled
    catch Error:
        RestoreVersionIni(cacheFolder, targetPath)
        return Error
```

### 3.2 构建当前文件哈希映射

```
function BuildFileHashMap(directory):
    map = {}   // hash -> absolutePath
    for each file in directory recursively:
        try:
            hash = SHA256(file)
            map[hash] = file
        catch:
            log warning
    return map
```

用途：
- 如果目标文件哈希已存在于当前目录，可直接复用，无需下载。
- 打补丁时用于定位旧文件路径。

### 3.3 差异清单选择

Release Manifest 有两种：

- **完整清单**：包含目标版本所有文件。
- **差量清单**：只包含从某个旧版本升级到新版本时需要变更的文件。

选择规则：

```
if releaseDetail.diffs contains currentVersion:
    use diff manifest
else:
    use full manifest
```

### 3.4 下载文件

对清单中每个文件，判断需要下载什么：

```
function DownloadNeededFiles(manifest, currentFiles, cacheFolder):
    tasks = []
    for file in manifest.files:
        if currentFiles contains file.hash:
            // 文件未变化，跳过
            continue

        if file.patch is not null:
            // 需要下载补丁包
            id = file.patch.id
            size = file.patch.patchSize
            hash = file.patch.patchHash
            suffix = file.patch.urlSuffix
        else:
            // 需要下载完整压缩包
            id = file.id
            size = file.compressedSize
            hash = file.compressedHash
            suffix = file.urlSuffix

        tasks.add({id, size, hash, suffix})

    tasks = deduplicate(tasks, by id)

    parallel for each task in tasks:
        DownloadWithRetry(task, cacheFolder)
```

### 3.5 下载单个文件（带断点续传）

```
function DownloadWithRetry(task, cacheFolder):
    path = cacheFolder + task.id
    for attempt in 1..maxRetry:
        try:
            DownloadResumable(task.url, path, task.expectedSize)
            hash = SHA256(path)
            if equalsIgnoreCase(hash, task.expectedHash):
                return
            else:
                Delete(path)
        catch error:
            log error
    throw ChecksumError
```

```
function DownloadResumable(url, path, expectedSize):
    open path as read/write stream, create if not exists
    existing = stream.length
    if existing >= expectedSize:
        return

    request = HTTP GET url
    request.header["Range"] = "bytes=" + existing + "-"
    response = send(request)
    if response has Content-Range:
        stream.position = Content-Range.from
    else:
        stream.position = 0

    while chunk = read(response.body):
        stream.write(chunk)
        updateProgress(chunk.length)
```

### 3.6 解压/打补丁与校验

```
function PatchAndVerifyFiles(manifest, currentFiles, targetPath, cacheFolder):
    parallel for each file in manifest.files:
        targetFile = targetPath + file.path
        CreateDirectory(parent(targetFile))

        if currentFiles contains file.hash:
            // 文件无变化，直接复制
            Copy(currentFiles[file.hash], targetFile)
        else if file.patch is null:
            // 下载的是完整压缩包，zstd 解压
            compressed = cacheFolder + file.id
            DecompressZstd(compressed, targetFile)
        else:
            // 使用 HPatch 合并旧文件 + patch
            oldFile = currentFiles[file.patch.oldFileHash]
            if oldFile exists and size == file.patch.oldFileSize:
                patchFile = cacheFolder + file.patch.id
                patchSlice = ReadSlice(patchFile, file.patch.offset, file.patch.length)
                HPatch(oldFile, patchSlice, targetFile)

        // 最终校验
        if SHA256(targetFile) != file.hash:
            Delete(targetFile)
            throw VerifyError
```

### 3.7 删除旧文件

```
function DeleteObsoleteFiles(manifest, targetPath):
    for path in manifest.deleteFiles:
        full = targetPath + path
        if exists(full):
            Delete(full)
```

### 3.8 version.ini 备份与回滚

- 更新前把 `targetPath/version.ini` 复制到 `cacheFolder/version.ini`。
- 更新失败或取消时，把备份复制回去。
- 这是最低限度的回滚，保证程序认为自己版本未变。

## 4. 启动器自身更新（Setup 版）

Setup 版不能自覆盖运行中的文件，因此采用“下载安装包，由安装包替换”的模式。

### 4.1 流程

```
function SelfUpdateSetup(releaseDetail):
    setupPath = cacheFolder + releaseDetail.setup.fileName
    if exists(setupPath) and SHA256(setupPath) == releaseDetail.setup.hash:
        // 已存在且校验通过
    else:
        DownloadResumable(releaseDetail.setup.url, setupPath, releaseDetail.setup.size)
        Verify SHA256

    LaunchAsAdmin(setupPath, [
        "update",
        "--InstallFolder", appDir,
        "--OldVersion", currentVersion,
        "--NewVersion", releaseDetail.version,
        "--Preview", enablePreview,
        "--pid", currentProcessId
    ])
```

### 4.2 安装包内部更新逻辑

安装包以管理员启动后，执行与便携版类似的逻辑：

```
function InstallerUpdate(installFolder, oldVersion, newVersion, preview):
    manifest = FetchManifest(oldVersion, newVersion, preview)
    SetAllFilesNormal(installFolder)

    if manifest.diffVersion is not null:
        DownloadDiffFiles(manifest, installFolder)

    VerifyAllFiles(manifest, installFolder)
    DeleteObsoleteFiles(manifest, installFolder)
```

### 4.3 全量校验逻辑

安装包更新时，不仅要下载差异，还要对清单中每个文件做最终校验：

```
function VerifyAllFiles(manifest, installFolder):
    parallel for each file in manifest.files:
        path = installFolder + file.path
        if not exists(path) or SHA256(path) != file.hash:
            Delete(path)
            DownloadZstd(manifest.urlPrefix + file.id, path, file.size)
            if SHA256(path) != file.hash:
                throw VerifyError
```

## 5. 游戏预下载与更新核心逻辑

游戏更新比启动器自身更新更复杂，因为需要处理多种下载模式、语音包、差量补丁、分块等。

### 5.1 数据分层

```
GameConfig          // 游戏配置，含默认下载模式、语音包路径、黑名单路径等
GameBranch          // 分支信息，含 Main / PreDownload 两个包
GamePackage         // 传统压缩包信息，含 Main / PreDownload
GameSophonChunkBuild    // Sophon 分块构建
GameSophonPatchBuild    // Sophon 差量补丁构建
```

### 5.2 操作类型

| 操作 | 含义 |
|------|------|
| Install | 全新安装 |
| Predownload | 预下载（维护前） |
| Update | 正式更新（维护后） |
| Repair | 修复（重新校验并补全） |

### 5.3 预下载流程

```
function PreDownload(gameId, installPath, audioLanguage):
    localVersion = ReadLocalGameVersion(installPath)
    if localVersion is null:
        return Unavailable

    config = FetchGameConfig(gameId)
    if config is null:
        return Unavailable

    // 选择预下载资源
    if config.defaultDownloadMode in [Chunk, LDIFF]:
        branch = FetchGameBranch(gameId)
        if branch.preDownload is null:
            return Unavailable

        canPatch = branch.preDownload.diffTags contains localVersion
        if canPatch:
            patchBuild = FetchSophonPatchBuild(branch, branch.preDownload)
        if patchBuild is null:
            chunkBuild = FetchSophonChunkBuild(branch, branch.preDownload)
    else:
        package = FetchGamePackage(gameId)
        if package.preDownload.major is null:
            return Unavailable

    // 计算大小
    size = CalculatePackageSize(localVersion, audioLanguage)

    // 用户确认后启动下载任务
    task = CreateTask(Predownload, gameId, installPath, audioLanguage)
    StartTask(task)
```

### 5.4 预下载任务执行

```
function ExecutePredownload(task):
    PrepareGamePackage(task)   // 生成 TaskFiles
    files = GetPredownloadFiles(task.taskFiles)

    task.totalBytes = sum(files.size)
    task.state = Downloading

    parallel for each file in files:
        DownloadWithRetry(file)

    // 标记预下载完成
    WriteConfigIni(task, predownload = "localVersion,predownloadVersion,audioLanguage")
    task.state = Finish
```

`GetPredownloadFiles` 的作用：把不同类型的任务文件统一抽象成“需要预先下载的文件”。

```
function GetPredownloadFiles(taskFiles):
    result = {}
    for file in taskFiles:
        if file.mode == SingleFile:
            add result[file.fullPath] = {path: file.fullPath, url: file.url, size: file.size, hash: file.md5}
        else if file.mode == CompressedPackage:
            for pkg in file.compressedPackages:
                add result[pkg.fullPath] = {path: pkg.fullPath, url: pkg.url, size: pkg.size, hash: pkg.md5}
        else if file.mode == Chunk:
            for chunk in file.chunks:
                if chunk.originalFileFullPath is empty:   // 本地没有可用 chunk
                    path = installPath + "/chunk/" + chunk.id
                    add result[chunk.id] = {path, url: chunk.url, size: chunk.compressedSize, hash: chunk.compressedMd5}
        else if file.mode == Patch and file.patch is not null:
            path = installPath + "/ldiff/" + file.patch.id
            add result[file.patch.id] = {path, url: file.patch.url, size: file.patch.patchFileSize, hash: file.patch.patchFileMd5}
    return values(result)
```

### 5.5 判断预下载是否完成

```
function IsPredownloadFinished(gameId, installPath):
    localVersion = ReadLocalGameVersion(installPath)
    predownloadInfo = ReadConfigIni(installPath, "predownload")
    if predownloadInfo is null:
        return false

    parts = split(predownloadInfo, ",")
    if parts.length != 3:
        return false

    storedLocalVersion = parts[0]
    storedPredownloadVersion = parts[1]
    storedAudioLanguage = parts[2]

    currentPredownloadVersion = FetchPredownloadVersion(gameId)
    currentAudioLanguage = GetAudioLanguage(gameId, installPath)

    return localVersion == storedLocalVersion
       and currentPredownloadVersion == storedPredownloadVersion
       and currentAudioLanguage == storedAudioLanguage
```

## 6. 游戏正式更新流程

### 6.1 更新准备

```
function PrepareUpdate(gameId, installPath, audioLanguage):
    localVersion = ReadLocalGameVersion(installPath)
    config = FetchGameConfig(gameId)

    task = CreateTask(Update, gameId, installPath, audioLanguage)
    task.localVersion = localVersion

    if config.defaultDownloadMode in [Chunk, LDIFF]:
        branch = FetchGameBranch(gameId)
        task.latestVersion = branch.main.tag

        canPatch = branch.main.diffTags contains localVersion
        if canPatch:
            task.patchBuild = FetchSophonPatchBuild(branch, branch.main)
        if task.patchBuild is null:
            task.chunkBuild = FetchSophonChunkBuild(branch, branch.main, "")
            task.localChunkBuild = FetchSophonChunkBuild(branch, branch.main, localVersion)
    else:
        package = FetchGamePackage(gameId)
        task.latestVersion = package.main.major.version
        task.package = package

    GenerateTaskFiles(task)
    return task
```

## 7. 下载模式执行细节

### 7.1 Patch 模式执行

```
function ExecutePatchUpdate(task):
    files = GetPredownloadFiles(task.taskFiles)
    task.totalBytes = sum(files.size)
    task.state = Downloading

    parallel for each file in files:
        DownloadWithRetry(file)

    task.state = Merging
    task.progressPercent = 0
    increment = 1.0 / task.taskFiles.length

    parallel for each taskFile in task.taskFiles:
        if taskFile.isFinished:
            task.progressPercent += increment
            continue
        PatchDiffFile(taskFile)
        taskFile.isFinished = true
        task.progressPercent += increment

    for file in task.deleteFiles:
        Delete(installPath + file.file)
```

```
function PatchDiffFile(taskFile):
    patchPath = installPath + "/ldiff/" + taskFile.patch.id
    patchSlice = ReadSlice(patchPath, taskFile.patch.patchOffset, taskFile.patch.patchLength)
    tmpPath = taskFile.fullPath + "_tmp"

    if taskFile.patch.originalFileFullPath is empty:
        // 无旧文件，直接生成
        if taskFile.patch.compression:
            DecompressZstd(patchSlice, tmpPath)
        else:
            Copy(patchSlice, tmpPath)
    else:
        // HPatch 合并
        HPatch(taskFile.patch.originalFileFullPath, patchSlice, tmpPath)

    Move(tmpPath, taskFile.fullPath)
    if originalFile != fullPath:
        Delete(originalFile)
```

### 7.2 Chunk 模式执行

```
function ExecuteChunkUpdate(task):

    task.state = Downloading

    parallel for each taskFile in task.taskFiles:

```

```
function DownloadChunksToFile(taskFile):
    if FileExistsAndMD5Match(taskFile.fullPath, taskFile.size, taskFile.md5):
        return

    tmpPath = taskFile.fullPath + "_tmp"
    OpenStream(tmpPath)

    for chunk in taskFile.chunks:
        if fs.length >= chunk.offset + chunk.uncompressedSize:
            // 此 chunk 已写入
            continue

        fs.position = chunk.offset

        if chunk.originalFileFullPath exists and size matches:
            // 从旧文件 slice 复用
            slice = ReadSlice(chunk.originalFileFullPath, chunk.originalFileOffset, chunk.uncompressedSize)
            if MD5(slice) == chunk.uncompressedMd5:
                Copy(slice, fs)
                continue

        chunkCache = installPath + "/chunk/" + chunk.id
        if FileExistsAndMD5Match(chunkCache, chunk.compressedSize, chunk.compressedMd5):
            // 从已下载缓存复用
            DecompressZstd(chunkCache, fs)
            continue

        // 从网络下载
        response = HTTP GET chunk.url
        Pipe: network -> zstd decompress -> fs
        UpdateProgress(downloadedBytes)

    CloseStream()
    if MD5(tmpPath) == taskFile.md5:
        Move(tmpPath, taskFile.fullPath)
    else:
        Delete(tmpPath)
        throw VerifyError
```

### 7.3 CompressedPackage 模式执行

```
function ExecutePackageUpdate(task):
    files = GetPredownloadFiles(task.taskFiles)
    task.totalBytes = sum(files.size)
    task.state = Downloading

    parallel for each file in files:
        DownloadWithRetry(file)

    task.state = Decompressing

        taskFile.isFinished = true
```

```
function ExtractCompressedPackage(taskFile):
    combinedStream = CombineFiles(taskFile.compressedPackages.map(p => p.fullPath))
    archive = Open7zArchive(combinedStream)

    ratio = taskFile.size / totalSize
    hasDiff = archive.contains("hdiffmap.json") or archive.contains("hdifffiles.txt")
    if hasDiff:
        extractRatio = ratio / 2
    else:
        extractRatio = ratio

    archive.extractTo(installPath, onProgress => update percent)
    task.progressPercent += extractRatio

    if hasDiff:
        PatchCompressedPackageDiffFiles(ratio / 2)
    task.progressPercent += ratio / 2
```

```
function PatchCompressedPackageDiffFiles(ratio):

        for item in diffMap.items:
```

## 8. 通用下载引擎设计

### 8.1 断点续传

所有大文件下载都应支持断点续传：

```
function ResumableDownload(url, path, expectedSize, progressCallback):
    stream = Open(path, readWrite, createIfNotExists)
    existing = stream.length

    if existing >= expectedSize:
        return

    request = HTTP GET url
    request.header["Range"] = "bytes=" + existing + "-"
    response = send(request)

    if response.status == 206 Partial Content:
        stream.position = response.contentRange.from
    else:
        stream.position = 0
        stream.truncate(0)

    while chunk = read(response.body):
        stream.write(chunk)
        progressCallback(chunk.length)
```

### 8.2 重试机制

```
function DownloadWithRetry(url, path, expectedSize, expectedHash, maxRetry):
    for attempt in 1..maxRetry:
        try:
            ResumableDownload(url, path, expectedSize)
            if Hash(path) == expectedHash:
                return
            Delete(path)
        except NetworkError, Timeout:
            sleep(linearBackoff(attempt))
    throw DownloadFailed
```

### 8.3 并发下载

- 文件级并行：多个文件同时下载。
- 注意：同一目标文件的写入不能并行，但不同文件可以。
- 进度更新使用线程安全的原子计数器。

### 8.4 限速

使用令牌桶算法：

```
class RateLimiter:
    tokens: float
    lastTime: timestamp
    maxTokens: float
    refillRate: tokens per second

    function Acquire(bytes):
        now = currentTime()
        elapsed = now - lastTime
        tokens = min(maxTokens, tokens + elapsed * refillRate)
        lastTime = now
        if tokens >= bytes:
            tokens -= bytes
            return true
        else:
            sleep((bytes - tokens) / refillRate)
            tokens = 0
            return true
```

## 9. 状态机与进度

### 9.1 更新状态机

```
Stop -> Pending -> Downloading -> Finish
                -> Error
                -> Stop (canceled)
                -> NotSupport
```

### 9.2 游戏任务状态机

```
Stop -> Waiting -> Downloading -> Decompressing/Merging/Verifying -> Finish
                              -> Paused
                              -> Error
                              -> Stop
```

同一游戏同一时间只能有一个任务在执行，其他任务进入 Queueing 状态。

### 9.3 进度计算

- **下载进度**：`finishDownloadBytes / totalDownloadBytes`
- **合并/解压进度**：`progressPercent`（0.0 ~ 1.0）
- **速度**：每秒采样一次 `finishBytes` 差值
- **剩余时间**：`(total - finish) / speed`

## 11. 跨语言实现建议

1. **并发模型**：使用语言原生线程池 / 协程，文件写入必须串行，网络下载可以并行。
2. **原子操作**：进度计数器使用原子整数，避免并发问题。
3. **临时文件**：所有下载先写入 `.tmp` / `_tmp`，校验通过后再 move/rename。
4. **异常处理**：网络异常只重试，文件校验失败必须删除并重新下载。
5. **取消机制**：通过 CancellationToken / Context 传递，取消时优雅停止并保留可续传文件。
6. **日志**：记录每个文件的下载、校验、合并、删除操作，便于排查。
7. **平台差异**：
   - Windows 下 Setup 更新需要管理员权限。
   - 硬链接仅在 NTFS 同一分区有效。
   - 文件属性处理（设为 Normal 避免只读错误）。
